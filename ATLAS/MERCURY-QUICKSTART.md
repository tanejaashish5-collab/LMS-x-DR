## Overview

MERCURY is the distribution agent that publishes landing pages to multiple channels (Reddit, email, Meta ads, Google ads) with channel-specific copy generated by Claude Haiku.

## Features

### 1. Channel-Specific Copy Generation
- Uses Claude Haiku (cheap & fast) for copy generation
- Generates optimized copy for each channel:
  - **Reddit**: Conversational, authentic posts for r/Entrepreneur, r/smallbusiness
  - **Email**: Professional B2B cold email outreach
  - **Meta Ads**: Facebook/Instagram ad copy with targeting
  - **Google Ads**: Search ad copy with keywords
- Cost: ~$0.01 per channel

### 2. Multi-Channel Distribution
- **Reddit**: Automated posting via PRAW (Python Reddit API Wrapper)
- **Email**: Sends via n8n webhook integration
- **Meta Ads**: Requires manual setup in Ads Manager
- **Google Ads**: Requires manual setup in Google Ads
- Simulated posting available when credentials not configured

### 3. VAULT Integration
- Requests budget approval before distribution
- Tracks all spending in database
- Estimated cost: $0.05 per distribution (copy generation only)

### 4. Distribution Tracking
- Logs all distributions to `atlas_agent_logs`
- Tracks channel performance
- Stores generated copy for future reference
- Updates experiment status to 'running'


## Configuration

### Reddit API Setup (Required for actual posting)

1. Create Reddit application at https://www.reddit.com/prefs/apps
2. Choose "script" type
3. Note your `client_id` and `client_secret`
4. Set environment variables:

```bash
export REDDIT_CLIENT_ID='your_client_id_here'
export REDDIT_CLIENT_SECRET='your_client_secret_here'
export REDDIT_USERNAME='your_reddit_username'
export REDDIT_PASSWORD='your_reddit_password'
export REDDIT_USER_AGENT='ATLAS-MERCURY/1.0'
```

5. Install PRAW library:

```bash
pip install praw
```

### n8n Email Webhook (Required for email distribution)

1. Create n8n workflow with webhook trigger
2. Configure email sending node
3. Set webhook URL:

```bash
export N8N_EMAIL_WEBHOOK='https://your-n8n.com/webhook/atlas-email'
```

### Meta Ads (Optional)

Meta ads require manual setup in Ads Manager. MERCURY generates copy but doesn't create campaigns automatically.

### Google Ads (Optional)

Google ads require manual setup in Google Ads. MERCURY generates copy but doesn't create campaigns automatically.

## Integration with ATLAS Workflow

```python

        self.mercury = MERCURYAgent(vault_agent=self.vault)

    async def launch_experiment(self, experiment_id: str):
        # Step 1: FORGE builds landing page
```

## Error Handling

MERCURY handles errors gracefully:
- Budget denied → Blocks all channels, returns budget_denied status
- Copy generation fails → Skips channel, logs error, continues with others
- Distribution fails → Logs error, continues with remaining channels
- Database errors → Logs warning, doesn't block distribution
- All errors logged to `atlas_agent_logs`

## Troubleshooting

### "Reddit credentials not configured - simulating post"
- This is expected when Reddit API credentials are not set
- Set environment variables and install PRAW to enable actual posting

### "Email sending via n8n webhook - simulating"
- This is expected when n8n webhook URL is not configured
- Set N8N_EMAIL_WEBHOOK environment variable to enable email sending

### "Copy generation failed: JSON decode error"
- Claude Haiku sometimes returns non-JSON text
- MERCURY falls back to text parsing automatically
- Check logs for actual error details

### "Budget denied by VAULT"
- VAULT rejected the budget request
- Check VAULT budget status
- Reduce number of distribution channels
- Wait for monthly budget refresh
